# 抖音视频收藏技能

当用户提供抖音视频链接（长链或短链）并表达收藏、下载或整理意图时，触发此技能。

## 1. 参数获取与路径决策
- **提取 URL**：从用户输入中精准提取抖音链接。
- **确定输出目录 (`--output-dir`)**：
    1. 优先使用用户在对话中显式指定的路径。
    2. 若用户未指定，使用 OpenClaw 配置中的 `default_output_dir`。
    3. 若以上均无，默认使用 `./favorites`。

## 2. 执行流程
1. **前置告知**：
    - 告知用户正在处理链接。
    - **特别提醒**：如果是首次运行，系统需要下载约 1.5GB 的 ASR 模型，耗时较长，请耐心等待。
2. **调用核心脚本**：
    - 执行命令：`python douyin-favorites.py <URL> --output-dir <CONFIRMED_DIR>`。
3. **读取转录结果**：
    - 脚本运行完成后，定位到输出目录下的 `./<视频ID>/` 目录
    - 读取 `<视频ID>.json` 文件，提取视频标题（title）、时常（duration）、视频ID（tiktok_id）、原始链接（original_url）。
    - 读取 `<视频ID>_raw.md` 文件所有内容，获得视频正文（video_body）。

4. **AI 总结与呈现**：
    - **生成总结**：基于获取到的视频标题（title）、视频正文（video_body），由 AI 自动生成一份简洁的内容摘要。
    - **总结格式**：将总结格式化为 Markdown 格式，包含视频标题、视频ID、原始链接、时常、内容摘要。
    - **总结文件**：将总结保存为输出目录下的 `./<视频ID>/<视频ID>_summary.md` 文件。
    - **最终交付**：向用户确认视频已保存，并展示“视频信息 + AI 总结 + 总结文件路径”。

## 3. 注意事项
- **硬件加速**：系统会自动检测 NVIDIA GPU 并在可用时通过 CUDA 加速识别。
- **故障处理**：若 `ffmpeg` 未安装或链接失效，请礼貌提示用户检查环境或链接。
- **处理反馈**：系统在开始收藏前给用户一个“处理中”的反馈。
